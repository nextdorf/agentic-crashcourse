# Crash Course Instructions

- These instructions apply to Markdown chapters in this directory.
- Chapters should work as short live-workshop material and as later self-study notes.
- Prefer a concrete demo, workflow, or example before abstract explanation.
- Keep main chapters focused on material that can realistically be covered in the workshop.
- Include practical artifacts where useful: prompts, commands, config snippets, file names, screenshots to discuss, or expected outcomes.
- Call out traps, tradeoffs, and failure modes when they affect how someone should use the tool.
- Move broad surveys, niche comparisons, provider opinions, and deep rabbit holes to appendix or further reading material.
